### 1.3 NIP-28 Public Chatチャンネル管理

NIP-28のkind一覧:
- `kind:40` チャンネル作成（メタデータ: name, about, picture）
- `kind:41` チャンネルメタデータ更新
- `kind:42` チャンネルメッセージ投稿
- `kind:43` チャンネルメッセージの非表示（モデレーション）
- `kind:44` チャンネルユーザーのミュート（モデレーション）

- [ ] vimrc読書会用の固定チャンネル定義
  - [ ] kind:40 でチャンネルを作成 or 既存チャンネルIDを設定ファイルで管理
  - [ ] チャンネルメタデータ（name, about, picture）の取得と表示
- [ ] チャンネルメッセージのフィルタ生成
  - [ ] kind:42 イベントのフィルタ（チャンネル作成イベントIDでのeタグ絞り込み）
  - [ ] eタグの先頭要素はチャンネル作成イベントID（NIP-28仕様）

## 依存関係

```
Phase 0（設計決定）
  ↓
Phase 1（基盤: リレー接続、認証、チャンネル管理）
  ↓
Phase 2（コア: 投稿、表示、ユーザー区別、Markdown、Permalink、閲覧、アカウント、プロフィール）
  ↓
Phase 3（推奨: 編集、アイコン）  ←  Phase 5（E2Eテスト）は Phase 2 完了後から開始可能
  ↓
Phase 4（追加: 削除、リアクション、返信、引用）
  ↓
Phase 6（リリース準備）
```

## 備考

- 各Phaseのタスクは `/vimrc-reading-implementer` がTDD（Red→Green→Refactor）で実装する
- 設計判断が必要な場面では `/vimrc-reading-architecture` に相談する
- 要件の追加・変更は `/vimrc-reading-requirements` で議論する
- ユニットテストは実装と同時にTDDで書く。E2Eテストは Phase 5 で `/vimrc-reading-e2e-tester` が書く
- マイクロコミット（Conventional Commits）で進める
